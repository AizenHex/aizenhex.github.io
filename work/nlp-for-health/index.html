NLP for Health — Triage otomatis dari catatan bahasa Indonesia

Model klasifikasi gejala pasien dari catatan perawat di puskesmas, menggunakan kombinasi fine-tuning IndoBERT dan aturan klinis sederhana.

2024

Latar Belakang

Di banyak puskesmas Indonesia, proses triage (penentuan prioritas pasien) masih sangat manual. Perawat menuliskan keluhan pasien dalam teks bebas — sering campur bahasa Indonesia baku, bahasa daerah, dan istilah medis singkat.

Pertanyaan yang saya ajukan: bisakah model NLP membantu memberi rekomendasi kategori triage (kritis / urgen / non-urgen) dari catatan ini, sebagai second opinion untuk perawat?

Kenapa Susah

Datasetnya ada, tapi tidak ideal:

  • ~8.000 catatan dari 3 puskesmas (anonim)
  • Label triage tidak selalu konsisten — bergantung perawat
  • Banyak noise: typo, singkatan informal, istilah lokal

Model pre-trained bahasa Indonesia yang ada (IndoBERT, IndoRoBERTa) di-training di Wikipedia & berita. Domainnya jauh dari catatan medis.

Pendekatan

Saya pakai dua-layer system:

1. Rule-based filter

Beberapa kata kunci medis — "henti napas", "kejang", "pendarahan massif" — selalu kritis. Saya encode sebagai regex + synonym list. Ini menangani ~15% kasus dengan akurasi mendekati 100%.

2. IndoBERT fine-tuned

Untuk sisanya, saya fine-tune IndoBERT pada dataset saya:

  • Preprocessing: normalisasi singkatan umum ("ph" → "pasien hamil", "tdk" → "tidak")
  • 3 epoch, learning rate 2e-5, batch 16
  • Augmentasi: back-translation ID → EN → ID untuk class minoritas (kritis)

Evaluasi

Saya ukur dengan:

  • Macro F1 — karena class imbalance (kritis hanya ~8%)
  • Recall @ kritis — karena false negative di sini bisa fatal
  • Human agreement — bandingkan output model vs 2 perawat senior

Target utama: recall kritis > 95%.

Hasil

  • Macro F1: 0.78
  • Recall kritis: 0.96
  • Precision kritis: 0.71 (trade-off yang disengaja — false positive lebih aman)
  • Disagreement dengan perawat senior: 11% kasus

11% disagreement itu awalnya mengecewakan, sampai saya sadari: perawat senior sendiri hanya setuju 83% satu sama lain. Model saya sudah dalam rentang variance manusia.

Deploy

Kami tidak deploy sebagai keputusan otomatis. Modelnya jalan sebagai suggestion pill di sidebar aplikasi perawat — "model menyarankan: urgen". Perawat bisa setuju atau tidak. Di 2 minggu pilot, perawat menggunakan suggestion ~60% kasus, dan melaporkan bahwa itu membantu terutama saat shift malam yang lelah.

Pelajaran

Medical NLP bukan chatbot. Konsekuensi kesalahan tinggi — setiap desain harus ditanya: "apa yang terjadi jika ini salah?"

Rule + ML > ML saja. Kombinasi memberi saya kontrol di kasus yang jelas dan fleksibilitas di kasus yang tidak jelas.

Label quality > model architecture. Saya habiskan 40% waktu di dataset cleaning. Itu waktu yang terbaik untuk dihabiskan.

Yang Saya Kerjakan Selanjutnya

  • Model multi-task: triage + diagnosis kasar
  • Suport bahasa daerah (Bali, Jawa) — masih kurang representasi
  • Integrasi dengan rekam medis elektronik resmi
